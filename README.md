# location-verification
Сервис для сравнения координат, полученных от Геокодера и DaData. 
Позволяет оценить расхождение в определении адресов.

## Возможности

- Геокодирование адреса через API Яндекса и DaData
- Вычисление расстояния между полученными координатами (в метрах, содержится в поле distanceMeters ответа)
- Сохранение истории запросов в MySQL
- REST API с валидацией входящих данных
- Обработка ошибок с возвратом понятных HTTP-статусов

## API

**POST /api/address**\
Позволяет сравнить координаты для переданного адреса.

Тело запроса (JSON):
```
json
{
  "address": "Москва, Красная площадь, 1"
}
```

Успешный ответ (200 OK):
```
json
{
  "address": "Москва, Красная площадь, 1",
  "yandexCoordinates": {
    "lat": 55.7539,
    "lon": 37.6208
  },
  "dadataCoordinates": {
    "lat": 55.754,
    "lon": 37.621
  },
  "distanceMeters": 16.74117440356108,
  "timestamp": "2026-06-27T20:34:37.263062100"
}
```

Ошибки:\
400 – невалидный адрес (менее 10 символов или отсутствует запятая)\
500 – внутренняя ошибка сервера\
502 – ошибка геокодирования (провайдер недоступен или вернул ошибку)

Тестирование\

```bash
mvn clean test
```

## Хранение данных

### Журналирование запросов

Сервис **автоматически сохраняет каждый входящий запрос** в базу данных, независимо от того, обрабатывался ли этот адрес ранее. \
Это реализовано как **журнал** всех обращений к API.

Для каждого запроса сохраняются:
- Входящий адрес.
- Координаты, полученные от Геокодера.
- Координаты, полученные от DaData.
- Расстояние между ними в метрах.
- Время создания записи.

Таким образом, таблица `address_comparison` содержит полную историю всех запросов, что позволяет:
- Анализировать динамику расхождений между геокодерами.
- Отслеживать изменения координат для одного адреса во времени.
- Аудировать использование API.

> **Важно:** Каждый новый запрос создаёт отдельную запись. Дедупликация не производится.

**Сущность `AddressComparison`** в пакете `entity` описывает структуру таблицы.


### Управление схемой базы данных
В проекте не используются миграционные инструменты (Flyway / Liquibase). 
Схема создаётся и обновляется автоматически через Hibernate, при старте приложения.
